### team rules

### Issue 제목 템플릿

[Type] 기능 - 이니셜

ex) [feat] 이민욱 자기소개 문서 작성

### branch 이름

type-#이슈번호/기능

ex) feat-#1/members-lmw

### commit 컨벤션 템플릿

[type #이슈번호] 기능 설명

ex) [feat #1] 이민욱 자기소개 문서 작성

`git commit -m "[feat #1] 이민욱 자기소개 문서 작성`

### 주의사항

- **브랜치 체크아웃 할때 `git pull` 한 번씩 진행하기**
- **본인이 작성하는 파일만 작성하기**
- **커밋할때 이슈 넘버 작성하기**
- **브랜치 기능에 맞는 기능만 구현하고 다른 기능이 필요하면 이슈 새로 작성**
- **`git add` 전에도 `git pull` 한번씩 진행**
